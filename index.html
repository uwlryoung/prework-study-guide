CSS

  • A . is a class selector (e.g. ".card" will select all classes titled "card")
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Display
  • Outer Display: How a Box is laid out in its environment
    • Outer-Display Type - Block:
      • The Box will break into a new line
      • The width and height properties are respected.
      • Padding, margin and border will cause other elements to be pushed away from the box
      • If width is not specified, the box will extend in the inline direction to fill the space available in its container. In most cases, the box will become as wide as its container, filling up 100% of the space available.
    • Out-Display Type - Inline:
      • The box will not break onto a new line.
      • The width and height properties will not apply.
      • Top and bottom padding, margins, and borders will apply but will not cause other inline boxes to move away from the box.
      • Left and right padding, margins, and borders will apply and will cause other inline boxes to move away from the box.
    • Inner Display: How elements inside the box are laid out
    • padding: the space between elements within a box
    • margin: the space around a box
  • Positions:
    • static: HTML elements are positioned this way by default, not affected by top, bottom, left or right properties. They are positioned according to the normal flow of the page.
    • relative: HTML elements are positioned relative to its normal position. This means you can set the top, right, bottom and left properties. Be careful: otehr content will not be adjusted to fit into any gap left by the element.
    • fixed: the HTML elements are fixed in one spot on the screen (as your scroll down, it will stay with you in that same spot). Use top, right, bottom and left properties to move it. A fixed element does not leave a gap and so it will overlap other items.
    • absolute: HTML elements are positioned relative to the nearest positioned ancestor. If there are no ancestors, is uses the document body and moves along with the page while scrolling. It can overlap elements.
    • sticky: HTML elements are positioned basd on the user's scroll position. This toggles between relative and fixed positions. it is positioned relative until a given offset position is met in the screen - then it sticks into place like fixed.

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: command to commit all files in the working branch
  • git commit -m "added code to HTML file": commits changes and -m " "associates a message with the commit
  • git pull origin main: pull the base branch into your feature branch before pushing up any code.
  • git push origin feature/[branch name]: pushes changes
  • git pull origin main: pulls in the latest changes

JavaScript

  • Make a variable by typing 'let [variable name];' You can also immediately set to be equal to a something by typing 'let [variable name] = [variable value];' Variable types:
  • Note: the above variable can also be created by using "var" instead of "let"
    • JavaScript is an object-orientated programming (OOP) lanugage
    • String: let myVariable = 'Bob'; (or "Bob") Quotes are needed.
    • Number: let myVariable = 10; Quotes not needed
    • Boolean: let myVariable = true; (or false)
    • Array: let myVariable = [1, 'Bob', 'Steve', 10]; (myVariable[0], myVariable[1], etc) - A way to store multiple values in a single reference
    • Object: Can be anything! Everything in JavaScript is an object and can be stored in a variable! let myVariable = document.querySelector('h1')
  • Variables are case sensitive! Be careful!
  • Use // to comment out one line, or /* */ to comment out many lines
  • Operators
    • Addition: + : Can add numbers or combine strings
    • Subtraction, Mltiplication, Division: - * / : As you would expect it to work
    • Assignment: = : Assigning variables
    • Strict Equality: === : Tests if two values arer equal and of the same data time and returns a true/false Boolean
    • Not, Does-not-equal: !, !== : Returns the logically opposite. When used with the == it tests whether they are equal or not and provides a Boolean
    • See here for even more operators!
  • For Loop: for ([set variable], [conditional], [variable update]){[code block]}
  • console.log([variable]): outputs to to the console/screen
  • Function - simply a set of instructions that tells the computer how to perform a certain task
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
